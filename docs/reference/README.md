# ElevenLabs reference — local snapshots

This directory holds a **local copy** of upstream reference material the CLI
tracks against. Checked-in so LLM contributors (Claude / Codex / Gemini /
Aider) can diff CLI behaviour against the upstream spec without a network
fetch, and so every audit has a reproducible baseline.

## Files

| File | Purpose | Refresh |
|---|---|---|
| `openapi.elevenlabs.json` | Full ElevenLabs OpenAPI 3.1 spec | `./refresh.sh` |
| `endpoints-inventory.txt` | Flat list: `METHOD /path [tag] operationId — summary`, one per line | Regenerated by `refresh.sh` |
| `cli-endpoints.txt` | Every `/v1/…` URL literal the CLI source hits, with the file it lives in | Regenerated by `refresh.sh` |

`refresh.sh` pulls the live spec from `https://api.elevenlabs.io/openapi.json`
and rebuilds the two derived inventories in place. Run it after any
upstream API change or before opening a coverage-audit PR.

## How to use these for a CLI coverage audit

```bash
# From the repo root:
./docs/reference/refresh.sh

# Compare CLI coverage to spec:
diff <(awk '{print $1, $2}' docs/reference/endpoints-inventory.txt | sort -u) \
     <(awk '{print $1, $2}' docs/reference/cli-endpoints.txt | sort -u)
```
